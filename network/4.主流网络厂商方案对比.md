## 交换机概述

交换机按照品牌可以分为品牌交换机、裸机交换机、白盒交换机。

- 品牌交换机厂商主要有 Cisco、Huawei、HPE 等；
- 裸机交换机主要是台湾企业，包括 Accton，Quanta QCT，Alpha Networks 和 Delta Computer 等公司，这些公司还是许多主流交换机供应商的原始设计制 造商（ODM）；
- 白盒交换机相当于你是打包购买了一个裸机交换机和一个操作系统，主要厂商有 Juniper、Arista、星网锐捷等。



| **类别**          | **核心构成**                 | **主要特点**                                                 | **典型厂商/供应商**                                          |
| ----------------- | ---------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **1. 品牌交换机** | **硬件 + 专有操作系统**      | 软硬件深度集成，提供完善的售后服务和技术支持。               | Cisco、Huawei (华为)、HPE 、华三、星网锐捷等。               |
| **2. 裸机交换机** | **纯硬件（Bare-Metal）**     | 主要提供原始设计制造 (ODM) 或原始设备制造 (OEM) 硬件，**不预装或捆绑操作系统**。 | Accton、Quanta QCT、Alpha Networks、Delta Computer 等台湾 ODM/OEM 企业。 |
| **3. 白盒交换机** | **裸机硬件 + 操作系统 (OS)** | 以**打包购买**的方式，将裸机交换机硬件与独立的网络操作系统进行整合。实现软硬件解耦 (Disaggregation)。 | Juniper、Arista、等（*注：这些厂商通常提供高度定制化的软硬件组合*）。 |





### 交换机芯片

交换芯片为交换机核心芯片之一，**决定了交换机的性能**。交换机主要功能是提供子网内的高性能和低延时交换，而高性能交换的功能主要由交换芯片完成。

同时由于交换机的部署 节点多、规模大，需要交换机具备更低的功耗、和更低的成本，对交换芯片功耗和成本提 出了更高的要求。



目前交换机芯片主要有商用和自研两种：

- 自研交换芯片：思科、瞻博和华为都有自研 400G 交换芯片
- 商用交换芯片：
  - 博通：TD 和 Tomahawk3/4，白盒大都选择 TH3/4 推出 400G 交换机；
  - Barefoot：可编程交换芯片 Tofino，被 Intel 收购;
  - Innovium：可编程交换芯片 Teralynx，初创公司;
  - 盛科网络：交换芯片 TransWarp 系列，由 CEC 和国家集成电产业基金共同投资。



## **厂商背景与市场定位**

思科、华为和华三是品牌网络设备领域的三大主流厂商，它们的交换机产品在技术实现、功能特性、操作系统和生态系统等方面存在显著差异

| **维度**     | **思科 (Cisco)**                   | **华为 (Huawei)**                | **华三 (H3C)**                     |
| ------------ | ---------------------------------- | -------------------------------- | ---------------------------------- |
| **起源**     | 美国厂商，网络设备行业领导者       | 中国厂商，全球通信设备龙头       | 最初是华为与3Com合资，现属紫光集团 |
| **市场定位** | 高端企业网、运营商市场             | 全场景覆盖（企业网、运营商、云） | 中国企业网市场主导者               |

## **关键差异总结**

1. **操作系统兼容性**
   - 思科`IOS/NX-OS`与其他两家不兼容，华为和华三的`VRP/Comware`命令相似度较高（约70%）。
   - **示例差异**：端口聚合协议思科用`LACP`，华为/华三用`lacp enable`。
2. **私有协议依赖**
   - 思科的`EIGRP`、`HSRP`可能导致迁移到华为/华三时需改用`OSPF`、`VRRP`。
3. **License策略**
   - 思科功能模块常需额外`License`（如SDWAN、加密），华为/华三基础功能更开放。
4. **运维工具链**
   - 思科：DNA Center、Prime
   - 华为：eSight、iMaster NCE
   - 华三：IMC智能管理中心



## **选型建议**

- **跨国企业/高预算**：优先思科（生态成熟，全球支持强）
- **国产化/性价比**：选华为（自主可控，5G融合能力强）
- **政府/教育行业**：华三（本土化服务好，符合等保要求）



## **迁移注意事项**

若需跨厂商替换设备，需重点关注：

1. 协议替代方案（如EIGRP→OSPF）
2. ACL语法转换（思科用编号，华为/华三支持命名ACL）
3. VLAN Trunk配置差异（思科`switchport trunk allowed vlan` vs 华为`port trunk allow-pass vlan`）
